# ExClipper

ExClipper는 몇 시간짜리 방송을 전부 직접 보지 않아도 되도록, AI가 먼저 하이라이트 후보를 찾고 사람이 짧은 후보만 확인하게 돕는 개인용 웹 편집 도구입니다.

- 제품 이름: **ExClipper**
- 저장소 이름: **`rettolight`**
- 배포 형태: **GitHub Pages 정적 웹앱**
- 배포 기준 경로: **`/rettolight/`**

저장소 이름과 Pages 주소는 기존 배포·북마크 호환을 위해 `rettolight`를 유지하고, 사용자에게 보이는 제품명과 새로 생성하는 산출물은 ExClipper를 사용합니다.
- 사용 대상: 컴퓨터나 영상 편집에 익숙하지 않은 스트리머·개인 편집자

## 지금 구현된 첫 수직 슬라이스

첫 구현은 사용자가 긴 영상을 보기 전에 앱이 먼저 후보를 제안하는 핵심 흐름을 실제 데이터로 연결합니다.

1. 사용자가 최대 12시간의 영상 파일을 고르면 앱이 브라우저 재생 가능 여부와 길이를 검사하고, 파일명·수정 시각과 무관한 시작·균등 중간·끝 내용 샘플 SHA-256 지문을 만듭니다. 12시간을 넘으면 분석 전에 파일을 나누도록 안내합니다.
2. 전용 Worker가 영상의 오디오를 순서대로 디코딩하면서 1초 단위 음량·피크·음성 대역·지속 시간을 계산합니다. 방송 안의 최근 약 2분 기준선과 비교해 평소보다 크게 튄 짧은 반응, 여러 초 이어지는 웃음·외침 가능성을 찾고 무음·한 번의 클릭음·계속 큰 배경음을 억제합니다. 원시 PCM은 남기지 않습니다.
3. 같은 방송의 CHZZK 채팅 JSON·JSONL·CSV 파일이 있다면 선택적으로 가져와 채팅량, 서로 다른 작성자 표기 수, 반응 표현, 반복·도배 패널티를 계산합니다. 작성자 표기는 가져온 파일 안의 비식별 키이며 실제 사람 수로 단정하지 않습니다. 채팅은 사건 뒤에 늦게 올라올 수 있어 반응 구간의 앞 문맥을 넉넉히 남깁니다.
4. 방송 오디오의 음성형·큰 반응 신호와 시청자 채팅을 후보를 만드는 **반응 anchor**로 사용합니다. 아직 스트리머 마이크와 게임·영상 소리를 분리하지 않으므로 후보 재생으로 실제 주체를 확인해야 합니다. 하루치 원본 한 개에서도 시간상 떨어진 반응은 서로 다른 여러 클립 후보가 됩니다. 희소 영상 장면 변화는 가까운 반응의 문맥·경계 근거로만 낮게 더하며, 반응 anchor가 전혀 없을 때도 화면 탐색 후보는 최대 2개·낮은 점수로 제한합니다. 겹친 후보만 억제하고 반응 정점 앞을 약 62.5% 남긴 30~60초 범위를 제안합니다.
5. 빠른 후보가 준비되면 별도 버튼을 기다리지 않고 AI 보강 단계가 이어집니다. 후보별 오디오와 대표 화면은 `gemini-3.5-flash`가 함께 해석하고, 최대 12시간 방송의 고른 음성 표본은 `qwen3-asr-flash-filetrans`가 한국어 시간순 챕터로 만듭니다. 그 챕터와 기존 후보를 `qwen3.7-plus`가 전체 맥락에서 다시 읽어 음악·오프닝·엔딩·반복 구간을 제외하고, 소리만으로 놓친 조용한 성취·사과·설정 회수 같은 의미 후보를 제안합니다. 새 의미 후보는 짧은 구간을 다시 확인한 뒤 Gemini 오디오·화면 검증까지 거칩니다.
6. Gemini가 돌려준 timestamp 대사는 반응 전·시점 부근·뒤의 최대 3개 확인 위치로 연결합니다. 모델이 제공하는 문장과 해석은 더 나은 검토 단서일 뿐 사실 확정이나 교정된 정확도가 아닙니다. 불분명한 말을 외국어처럼 지어내지 말고 생략하거나 `[불명]`으로 남기도록 요청하며, 앱도 길이·시간 범위·허용 필드를 다시 검사합니다. 사용자는 버튼을 눌러 실제 장면을 확인하고, 조정한 클립 범위 밖의 위치는 비활성화합니다.
7. `반응 종류 AI로 확인`을 누르면 별도 Worker가 후보마다 반응 정점 전·중·후의 약 10초 창을 최대 3개만 듣고, 고정 버전의 Audio Spectrogram Transformer로 웃음·고함/외침·비명·박수/환호 단서를 최대 2개까지 찾습니다. 첫 실행에는 모델 약 91MB와 WASM·Worker 실행 파일 약 23MB를 추가로 받을 수 있습니다. 스트리머 마이크와 게임·영상 소리를 분리하지 않은 혼합 오디오 결과라서, 사건이나 반응 주체를 확정하지 않고 반드시 재생 확인 위치와 한계를 함께 표시합니다.
8. 후보가 둘 이상이면 `AI 추천 순서 만들기`를 눌러 최대 12개 후보의 보존된 오디오·채팅·화면 근거를 반응 중심으로 다시 조합할 수 있습니다. 모든 후보를 빠짐없이 끝낸 반응 종류 분석만 공정하게 보강하고 Gemini 대사 문구는 점수에 넣지 않습니다. 먼저 이전→추천 순서와 쉬운 이유를 보여 주고, 사용자가 `추천 순서 적용`을 눌러야 카드 순서만 바뀝니다. 한 번에 되돌릴 수 있고 승인·제외·구간·재생 위치와 시간순 다운로드는 바뀌지 않습니다.
9. 각 후보 카드에서 빠른 분석·채팅·화면·Gemini 대사·반응 종류 단서를 `사건 단서 / 반응 단서 / 확인할 점`으로 합쳐 보여 줍니다. 사건·행위자·원인·결과를 확정하지 않고, 혼합 방송 오디오와 Gemini 결과는 반드시 주체·내용 확인이 필요한 모델 단서로 표시합니다. 가장 강한 반응 종류 창, 대사 위치, 반응 정점 순으로 `AI가 짚은 위치` 하나를 골라 곧바로 재생할 수 있습니다.
10. 분석 결과와 coverage가 IndexedDB에 기록되고 다시 읽어 동일성이 확인된 뒤, 실행당 하나뿐인 write-once terminal disposition까지 확정되어야 완료로 표시합니다. 같은 terminal의 동일한 재시도만 허용하고 완료·실패 결과가 서로 덮어쓰지 못합니다. 사용자는 내장 플레이어로 짧은 후보만 확인해 승인하거나 제외합니다.
11. 각 후보의 시작·끝을 5초씩 앞뒤로 움직이거나 현재 재생 위치로 지정할 수 있습니다. AI 원안은 보존되고, 여러 후보의 조정은 서로 섞이지 않습니다.
12. 승인한 후보가 하나 이상이면 최종 조정된 `HH:MM:SS` 시작·끝 시간을 화면에서 확인하고 복사하거나, Excel용 UTF-8 CSV·읽기 좋은 Markdown·개인정보를 뺀 JSON으로 받을 수 있습니다. JSON은 AI 제안과 사용자 최종 구간을 구분합니다.
13. 새로고침 뒤에는 완료 terminal을 목록의 유일한 기준으로 삼아 manifest·최종 결과·산출물 ID·입력 서명·coverage를 다시 맞춘 결과만 `지난 AI 분석 결과`로 엽니다. 원본을 다시 고르면 내용 샘플 지문·크기·길이가 맞을 때만 미리보기를 연결합니다.

현재 `0.3.28` 엔진은 **빠른 반응 탐색 → 후보 오디오·화면 검증 → 방송 전체 음성 챕터 → 전체 맥락 선별·의미 후보 보충**을 자동 단계로 수행합니다. 빠른 탐색은 이전에 정상 검출한 칼국수·껍데기·두바이초콜릿 계열의 음성형 반응을 유지하되 반복되는 음악형 신호가 후보가 되는 것을 막습니다. 전체 맥락 단계는 후보가 0개인 방송도 정상 결과로 인정하며, 마크 릴레이처럼 의미 있는 독립 사건이 없으면 억지로 후보를 만들지 않습니다. 각 유료 결과는 입력 서명과 모델 revision을 붙여 IndexedDB에 저장하고 새로고침·재연결 때 재사용합니다.

## 저장과 계정

현재 앱 버전은 `0.3.28`입니다. 후보별 Gemini 해석, Qwen 방송 전사 챕터, 전체 맥락 판정과 의미 후보 재확인 결과를 입력 서명별로 저장해 새로고침 뒤 복원합니다.

ExClipper는 회원가입이나 팀 공유 기능이 없는 **1인용 편집 어시스턴트**입니다.

- 선택한 원본 파일 자체, 원시 오디오, 전사 원문과 채팅 원문은 저장하지 않습니다. 분석 입력의 내용 샘플 지문·기능 정보와 후보의 오디오·채팅·시각 신호 종류, 시간, 점수, 허용된 집계 숫자만 브라우저 IndexedDB에 확정 기록합니다.
- 영속 저장은 레코드 종류별 허용 목록을 사용합니다. 원시 MIME·확장자, 채팅 원문·닉네임, 제한되지 않은 임의 문장은 저장 경계에서 거부하고, Gemini 해석은 후보별 허용 필드와 길이로 검증한 요약만 회차별로 보존합니다.
- 사용자가 고른 `File`로 분석과 미리보기를 진행하며, 페이지를 닫거나 원본을 바꾸면 다시 선택해야 합니다.
- 검증된 완료 분석은 새로고침 뒤 다시 열 수 있습니다. 빠른 분석뿐 아니라 후보별 Gemini 대사·사건·반응 해석도 실행 중 IndexedDB에 회차별로 저장해 후보 카드에서 이어서 볼 수 있습니다. 승인·제외 판단까지 복원하는 전체 프로젝트 저장은 아직 아닙니다. 다시 열면 후보는 모두 `검토 전`으로 시작합니다.
- 승인·제외 판단, 후보 시작·끝 조정, 반응 종류 AI 또는 추천 검토 순서는 아직 저장되지 않았다는 안내를 계속 보여 주며, 다른 영상·채팅·저장 결과로 이동하거나 페이지를 닫기 전 확인합니다. Gemini 대사·사건·반응 해석은 실행 중 IndexedDB에 회차별로 저장되지만 CSV·Markdown·JSON·복사 결과에는 포함되지 않습니다. 분석 버튼을 누른 즉시 시작 fence를 세우고 원본·채팅·시간 보정을 잠가 실행과 화면 입력이 섞이지 않게 합니다.
- 여러 사람이 같은 Pages 주소를 열 수는 있지만 서로의 프로젝트를 보거나 함께 편집하지 않습니다.
- 계정, 팀 공유, 프로젝트 공동 편집, 기기간 자동 동기화는 제공하지 않습니다.
- 이후 프로젝트 저장을 추가할 때는 브라우저 기록이 지워질 상황에 대비해 사용자가 직접 백업 파일을 내려받는 복구 경로를 함께 제공합니다.

Gemini와 Qwen API 키는 Cloudflare Worker Secret으로 설정하며 저장소·정적 빌드·브라우저 저장소·로그에는 넣지 않습니다. 사용자는 키를 입력하지 않으며 빠른 후보가 준비되면 AI 보강 단계가 자동으로 이어집니다.

## CHZZK 채팅

같은 방송의 CHZZK 채팅 기록이 있다면 선택적으로 가져와 시청자 반응을 실제 하이라이트 신호로 활용합니다.

- 구현된 경로: 사용자가 가진 JSONL·JSON·CSV 채팅 기록 가져오기
- 기본 개인정보 보호: 닉네임은 즉시 비식별화하고, 내보내기에는 원문과 닉네임을 포함하지 않음
- 장애 경계: 채팅 Worker가 없거나 60초 안에 응답하지 않으면 영상 결과는 보존하고, 사전 고지된 정책과 gap 근거를 기록한 `채팅 제외 완료`로 표시
- 이후 단계: 선택형 라이브 채팅 동반 수집기

공용 채팅 수집 백엔드나 문서화되지 않은 과거 채팅 조회 기능은 만들지 않습니다.

## 개발 서버에서 실행하기

[Node.js](https://nodejs.org/)의 현재 LTS 버전과 npm이 필요합니다. 저장소를 내려받은 뒤 터미널에서 다음 명령을 순서대로 실행합니다.

```bash
npm install
npm run dev
```

터미널에 표시된 개발 주소를 브라우저에서 열면 됩니다. 개발 서버를 끝내려면 터미널에서 `Ctrl+C`를 누릅니다.

검사와 테스트(typecheck, lint, unit test):

```bash
npm run check
```

배포용 파일 만들기:

```bash
npm run build
```

완성된 정적 파일은 `dist/` 폴더에 만들어집니다.

## GitHub Pages 배포

Vite의 공개 기준 경로는 저장소 이름에 맞춰 `/rettolight/`를 사용합니다. `main` 브랜치에 승인된 변경이 들어오면 GitHub Actions가 다음 순서로 검사하고 Pages용 artifact를 만듭니다.

1. `npm ci`
2. `npm run check`
3. `npm run build`
4. `dist/` 업로드
5. GitHub Pages 배포

워크플로 파일을 추가했다고 Pages가 자동으로 공개되는 것은 아닙니다. 저장소 관리자가 GitHub의 **Settings → Pages**에서 배포 원본을 **GitHub Actions**로 선택해야 합니다.

일반적인 배포 주소는 다음 형태입니다.

```text
https://<github-user>.github.io/rettolight/
```
