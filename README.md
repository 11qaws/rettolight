# Retto Highlight

Retto Highlight는 몇 시간짜리 방송을 전부 직접 보지 않아도 되도록, AI가 먼저 하이라이트 후보를 찾고 사람이 짧은 후보만 확인하게 돕는 개인용 웹 편집 도구입니다.

- 제품 이름: **Retto Highlight**
- 저장소 이름: **`rettolight`**
- 배포 형태: **GitHub Pages 정적 웹앱**
- 배포 기준 경로: **`/rettolight/`**
- 사용 대상: 컴퓨터나 영상 편집에 익숙하지 않은 스트리머·개인 편집자

## 지금 구현된 첫 수직 슬라이스

첫 구현은 사용자가 긴 영상을 보기 전에 앱이 먼저 후보를 제안하는 핵심 흐름을 실제 데이터로 연결합니다.

1. 사용자가 자신의 로컬 영상 파일을 고르면 앱이 브라우저 재생 가능 여부와 길이를 검사하고, 파일명·수정 시각과 무관한 시작·균등 중간·끝 내용 샘플 SHA-256 지문을 만듭니다.
2. 전용 Worker가 영상의 오디오를 순서대로 디코딩하면서 1초 단위 음량·피크·음성 대역·지속 시간을 계산합니다. 방송 안의 최근 약 2분 기준선과 비교해 평소보다 크게 튄 짧은 반응, 여러 초 이어지는 웃음·외침 가능성을 찾고 무음·한 번의 클릭음·계속 큰 배경음을 억제합니다. 원시 PCM은 남기지 않습니다.
3. 같은 방송의 CHZZK 채팅 JSON·JSONL·CSV 파일이 있다면 선택적으로 가져와 채팅량, 고유 참여자 수, 반응 표현, 반복·도배 패널티를 계산합니다. 채팅은 사건 뒤에 늦게 올라올 수 있어 반응 구간의 앞 문맥을 넉넉히 남깁니다.
4. 스트리머 오디오와 시청자 채팅을 후보를 만드는 **반응 anchor**로 사용합니다. 희소 영상 장면 변화는 가까운 반응의 문맥·경계 근거로만 낮게 더하며, 반응 anchor가 전혀 없을 때도 화면 탐색 후보는 최대 2개·낮은 점수로 제한합니다. 겹친 후보를 억제하고 반응 정점 앞을 약 62.5% 남긴 30~60초 범위를 제안합니다.
5. 분석 결과와 coverage가 IndexedDB에 기록되고 다시 읽어 동일성이 확인된 뒤, 실행당 하나뿐인 write-once terminal disposition까지 확정되어야 완료로 표시합니다. 같은 terminal의 동일한 재시도만 허용하고 완료·실패 결과가 서로 덮어쓰지 못합니다. 사용자는 내장 플레이어로 짧은 후보만 확인해 승인하거나 제외합니다.
6. 승인한 후보가 하나 이상이면 `HH:MM:SS` 시작·끝 시간을 화면에서 확인하고 복사하거나, Excel용 UTF-8 CSV·읽기 좋은 Markdown·개인정보를 뺀 JSON으로 받을 수 있습니다.
7. 새로고침 뒤에는 완료 terminal을 목록의 유일한 기준으로 삼아 manifest·최종 결과·산출물 ID·입력 서명·coverage를 다시 맞춘 결과만 `지난 AI 분석 결과`로 엽니다. 원본을 다시 고르면 내용 샘플 지문·크기·길이가 맞을 때만 미리보기를 연결합니다.

현재 `0.3.0` 엔진은 **스트리머의 오디오 반응을 우선하고, 선택적 채팅 반응을 결합하며, 영상 변화는 문맥으로만 쓰는 로컬 fast pass**입니다. 각 후보에는 `무슨 일이 있었나`, `스트리머 반응`, `시청자 반응`, `왜 볼 만한가`를 나눠 보여 줍니다. 다만 아직 대사를 받아쓰거나 화면 사건을 의미적으로 분류하지 않으므로, 구체적인 사건명은 꾸며내지 않고 `신호 기반 추정`이라고 표시합니다. 지원하지 않는 링크를 분석한 것처럼 표시하거나 임의 후보·가짜 확률을 만들지 않습니다. 다음 핵심 단계는 상위 후보에만 로컬 한국어 전사와 소형 음향 사건 모델을 적용해 사건·원인 설명과 경계를 더 정확히 만드는 것입니다.

## 영상과 개인정보

Retto Highlight는 회원가입이나 공유 서버가 없는 **1인용 로컬 우선 앱**입니다.

- 선택한 영상·음성은 외부 분석 서버로 업로드하지 않습니다.
- 선택한 원본 파일 자체, 원시 오디오, 전사 원문과 채팅 원문은 저장하지 않습니다. 분석 입력의 내용 샘플 지문·기능 정보와 후보의 오디오·채팅·시각 신호 종류, 시간, 점수, 허용된 집계 숫자만 브라우저 IndexedDB에 확정 기록합니다.
- 영속 저장은 레코드 종류별 허용 목록을 사용합니다. 임의 후보 문장, 원시 MIME·확장자, 채팅 원문·닉네임처럼 다른 내용을 숨길 수 있는 필드는 저장 경계에서 거부하고, 후보 설명은 화면에서 신호 종류로 다시 만듭니다.
- 현재 탭에서는 사용자가 고른 `File`을 직접 참조해 분석과 미리보기를 수행하며, 탭을 닫으면 그 파일 접근은 사라집니다.
- 검증된 완료 분석은 새로고침 뒤 다시 열 수 있습니다. 다만 현재는 **AI 분석 결과 복원**이며 승인·제외 판단까지 복원하는 전체 프로젝트 저장은 아닙니다. 다시 열면 후보는 모두 `검토 전`으로 시작합니다.
- 승인·제외 판단을 시작하면 아직 저장되지 않았다는 안내를 계속 보여 주며, 다른 영상·채팅·저장 결과로 이동하거나 페이지를 닫기 전 확인합니다. 분석 버튼을 누른 즉시 시작 fence를 세우고 원본·채팅·시간 보정을 잠가 실행과 화면 입력이 섞이지 않게 합니다.
- 여러 사람이 같은 Pages 주소를 열 수는 있지만 서로의 프로젝트를 보거나 함께 편집하지 않습니다.
- 계정, 팀 공유, 원격 프로젝트 데이터베이스, 기기간 자동 동기화, 클라우드 AI는 제공하지 않습니다.
- 이후 프로젝트 저장을 추가할 때는 브라우저 기록이 지워질 상황에 대비해 사용자가 직접 백업 파일을 내려받는 복구 경로를 함께 제공합니다.

AI 모델 구현 뒤에는 모델 파일을 인터넷에서 내려받을 수 있지만, 사용자가 선택한 원본 영상 자체를 모델 제공 서버로 보내지는 않습니다.

## CHZZK 채팅

같은 방송의 CHZZK 채팅 기록이 있다면 선택적으로 가져와 시청자 반응을 실제 하이라이트 신호로 활용합니다.

- 구현된 경로: 사용자가 가진 JSONL·JSON·CSV 채팅 기록 가져오기
- 기본 개인정보 보호: 닉네임은 즉시 비식별화하고, 내보내기에는 원문과 닉네임을 포함하지 않음
- 장애 경계: 채팅 Worker가 없거나 60초 안에 응답하지 않으면 영상 결과는 보존하고, 사전 고지된 정책과 gap 근거를 기록한 `채팅 제외 완료`로 표시
- 이후 단계: 사용자 컴퓨터에서만 실행되는 선택형 로컬 라이브 채팅 수집기

공용 채팅 수집 백엔드나 문서화되지 않은 과거 채팅 조회 기능은 만들지 않습니다.

## 로컬에서 실행하기

[Node.js](https://nodejs.org/)의 현재 LTS 버전과 npm이 필요합니다. 저장소를 내려받은 뒤 터미널에서 다음 명령을 순서대로 실행합니다.

```bash
npm install
npm run dev
```

터미널에 표시된 로컬 주소를 브라우저에서 열면 됩니다. 개발 서버를 끝내려면 터미널에서 `Ctrl+C`를 누릅니다.

검사와 테스트(typecheck, lint, unit test):

```bash
npm run check
```

배포용 파일 만들기:

```bash
npm run build
```

완성된 정적 파일은 `dist/` 폴더에 만들어집니다.

## GitHub Pages 배포

Vite의 공개 기준 경로는 저장소 이름에 맞춰 `/rettolight/`를 사용합니다. `main` 브랜치에 승인된 변경이 들어오면 GitHub Actions가 다음 순서로 검사하고 Pages용 artifact를 만듭니다.

1. `npm ci`
2. `npm run check`
3. `npm run build`
4. `dist/` 업로드
5. GitHub Pages 배포

워크플로 파일을 추가했다고 Pages가 자동으로 외부에 활성화되는 것은 아닙니다. 저장소 관리자가 GitHub의 **Settings → Pages**에서 배포 원본을 **GitHub Actions**로 선택해야 합니다.

일반적인 배포 주소는 다음 형태입니다.

```text
https://<github-user>.github.io/rettolight/
```

## 설계 문서

- [`PRODUCT_PLAN.md`](./PRODUCT_PLAN.md): 제품·UX·AI 분석·플랫폼 계획
- [`STATE_LIFECYCLE.md`](./STATE_LIFECYCLE.md): 상태, 이벤트, 허용 전이와 복구 계약
- [`OPERATIONS.md`](./OPERATIONS.md): Pages 배포, 백업, 롤백과 장애 대응
- [`DEVELOPMENT_LOG.md`](./DEVELOPMENT_LOG.md): 조사, 결정, 변경과 검증 기록

현재는 구현 초기 단계입니다. 장시간 파일을 전체 메모리에 올리지 않는 오디오 순차 분석, 스트리머 반응 우선 후보, 선택적 채팅 반응 결합, 낮은 비중의 영상 문맥 신호, 후보별 구조화 설명, 미리보기·검토, 검증된 로컬 결과 저장, 완료 분석 다시 열기, 같은 원본의 내용 샘플 지문 재연결, 타임코드 복사와 CSV·Markdown·JSON 결과 받기는 실제로 동작합니다. 학습된 음향 사건 분류, 한국어 대사 전사·사건 의미 해석, 실제 클립 파일 자르기·렌더, 후보 경계 보정, 승인·제외 판단까지 포함한 전체 프로젝트 복원, 전체 파일 바이트 해시, 선택형 로컬 라이브 채팅 수집기는 아직 제공하지 않습니다. 위 문서의 단계별 완료 조건을 기준으로 구현 상태를 판단해 주세요.
