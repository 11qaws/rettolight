## Answer

Expanded from original query via vocab: [highlight, visual, luma, frame, sampling, chat, score, fusion, audio, stream, robust]. The current engine is a deterministic visual-chat fast pass, not a streamer-reaction detector. selectVisualHighlightsFromSamples uses 32x18 global luma differences and sparse sampling; selectChatHighlights finds audience message bursts and reaction-token bursts; fuseHighlightCandidates normalizes each modality by within-signal rank/MAD and gives paired signals a bonus. These are useful discovery proxies, but visual cuts should be boundary/event evidence rather than primary highlight evidence. The next priority is full-timeline local audio reaction features, then selective Korean speech analysis, chat-delay alignment, and optional facecam motion support without facial-emotion labels.
